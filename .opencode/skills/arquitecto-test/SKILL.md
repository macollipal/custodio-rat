1. Arquitecto de Software (Prioridad Alta)

Objetivo:

Revisar arquitectura.
Detectar deuda técnica.
Proponer refactorizaciones.
Revisar escalabilidad.

Debe analizar:

Estructura de carpetas.
Separación de responsabilidades.
Patrones de diseño.
Acoplamiento.
Dependencias.

Entregables:

Informe arquitectónico.
Riesgos.
Roadmap técnico.
2. QA Lead / Test Architect (Prioridad Muy Alta)

Este sería el siguiente que implementaría.

Objetivo:

Encontrar bugs.
Encontrar casos borde.
Encontrar regresiones.

Debe generar:

Casos de prueba

Ejemplo:

Login

Usuario válido
Usuario inválido
Contraseña incorrecta
Token expirado
Usuario deshabilitado
Casos límite
Campos vacíos
Strings largos
Caracteres especiales
Carga masiva
Riesgos
Seguridad
Datos inconsistentes
Errores de flujo
3. Product Auditor

Este es muy útil para Custodio.

Debe responder:

¿Los menús tienen sentido?
¿La navegación es intuitiva?
¿Qué funcionalidades faltan?
¿Qué funcionalidades agregan más valor?

Piensa como cliente.

4. Compliance Auditor (Ley 21.719)

Muy importante para tu proyecto.

Debe revisar:

RAT.
ARCO.
Brechas.
Consentimientos.
Auditoría.

Y mapear todo contra la ley.

5. DBA Senior

Para revisar:

Índices.
Relaciones.
Consultas lentas.
Normalización.

A veces encuentra problemas que no se ven hasta que tienes decenas de clientes.

6. Security Auditor

Revisión de:

JWT.
RBAC.
OWASP Top 10.
Logs.
Secretos.
Permisos.
Mi combinación para Custodio Beta

Si tuviera que elegir solo 3 skills para esta fase:

1° Arquitecto de Software

Pregunta:

¿La plataforma puede crecer?

2° QA Lead / Test Architect

Pregunta:

¿Qué se rompe?

3° Compliance Auditor Ley 21.719

Pregunta:

¿Cumple realmente con la regulación?

Si detectas errores de arquitectura, diseño o configuración, indícalos aunque no estén relacionados.
